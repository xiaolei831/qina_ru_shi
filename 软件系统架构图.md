## 图2-5 软件系统架构图（ROS2 功能包 / 话题 / Action 关系）

本系统基于 ROS2 构建，按功能分层组织为「传感器驱动层 → 定位导航层 → 任务决策层 → 视觉抓取层 →
人机交互与云端层」，各功能包之间通过话题（Topic）与动作（Action）解耦通信。

```mermaid
flowchart TB
    %% ================= 传感器与执行层 =================
    subgraph DRV["传感器驱动 / 底盘执行层"]
        direction LR
        LIDAR["p2117_ros<br/>(oradar_lidar 雷达驱动)"]
        CAM["ros2_astra_camera<br/>(Astra 深度相机驱动)"]
        BASE["communication_base<br/>(底盘串口/里程计/IMU)"]
    end

    %% ================= 定位与导航层 =================
    subgraph NAV["定位与导航层"]
        direction LR
        SLAM["slam_cartgorpher<br/>(Cartographer 建图)"]
        LOC["jie_ware / lidar_loc<br/>(激光定位, map->odom)"]
        NAV2["nav2_qing<br/>(Nav2 导航栈 + 地图 + 语义点位)"]
    end

    %% ================= 任务决策层 =================
    subgraph TASK["任务决策 / 人机交互层"]
        direction LR
        VNC["voice_nav_control<br/>(语音任务状态机)"]
        AI["robot_ai<br/>(大模型语音问答 小青)"]
    end

    %% ================= 视觉抓取层 =================
    subgraph ARM["视觉识别与机械臂抓取层"]
        direction TB
        VT["visual_tracker<br/>(BPU 药品检测)"]
        CL["car_location_color<br/>(底盘对准)"]
        AP["direct_arm_pick_and_put<br/>(机械臂抓取/放置)"]
    end

    %% ================= 监控与云端层 =================
    subgraph CLOUD["监控与云端层"]
        direction LR
        WEB["robot_car_web<br/>(Web 状态监控)"]
        ONENET["onenet_bridge<br/>(OneNET 云端上报)"]
    end

    %% ---------- 传感器数据流 ----------
    LIDAR -- "/scan" --> LOC
    LIDAR -- "/scan" --> SLAM
    BASE -- "/odom /imu/data_raw" --> LOC
    BASE -- "/imu/data" --> SLAM
    CAM -- "/camera/color/image_raw<br/>/camera/depth/image_raw" --> VT

    %% ---------- 定位/导航 ----------
    SLAM -. "离线生成地图" .-> NAV2
    LOC -- "map->odom_combined (TF)" --> NAV2
    NAV2 -- "/cmd_vel" --> BASE

    %% ---------- 任务决策 ----------
    VNC == "Action: navigate_to_pose<br/>(NavigateToPose)" ==> NAV2
    VNC -- "/goal_pose" --> NAV2
    VNC -- "/target_medicine<br/>/medicine_pick_enable" --> VT
    VNC -- "/arm_state=rotate_put/no_msg" --> AP
    AP -- "/medicine_pick_state<br/>/arm_phase" --> VNC

    %% ---------- 视觉抓取内部 ----------
    VT -- "/color_position" --> CL
    VT -- "/color_position" --> AP
    CL -- "/cmd_vel" --> BASE
    CL -- "/arm_state=pick" --> AP
    AP -- "arm_controller / hand_controller<br/>joint_trajectory" --> BASE
    AP -- "/car_command" --> CL

    %% ---------- 监控与云端 ----------
    BASE -- "/PowerVoltage /imu/data_raw<br/>/ultrasonic_array 等" --> WEB
    NAV2 -- "/map /amcl_pose /odom_combined" --> WEB
    VNC -- "/voice_nav_task<br/>/medicine_delivery_record" --> WEB
    VNC -- "/medicine_delivery_record" --> ONENET
    BASE -- "/odom_combined /PowerVoltage" --> ONENET
    ONENET <-. "MQTT" .-> IOT(("OneNET 云平台"))
    WEB <-. "HTTP/WebSocket" .-> PC(("PC / 手机浏览器"))

    classDef drv fill:#E3F2FD,stroke:#1976D2,color:#0D47A1;
    classDef nav fill:#E8F5E9,stroke:#388E3C,color:#1B5E20;
    classDef task fill:#FFF3E0,stroke:#F57C00,color:#E65100;
    classDef arm fill:#F3E5F5,stroke:#8E24AA,color:#4A148C;
    classDef cloud fill:#FCE4EC,stroke:#C2185B,color:#880E4F;
    class LIDAR,CAM,BASE drv;
    class SLAM,LOC,NAV2 nav;
    class VNC,AI task;
    class VT,CL,AP arm;
    class WEB,ONENET cloud;
```

## 图2-6 送药任务执行流程图（任务状态机时序）

配合架构图使用，展示语音下达送药指令后各功能包的协作时序。

```mermaid
sequenceDiagram
    autonumber
    participant U as 用户/语音模块
    participant V as voice_nav_control
    participant N as nav2_qing (导航)
    participant T as visual_tracker (BPU识别)
    participant C as car_location_color (对准)
    participant A as direct_arm_pick_and_put (机械臂)
    participant R as robot_car_web / onenet_bridge

    U->>V: 串口指令 (取某药 + 送某病房)
    V->>N: Action navigate_to_pose → 药房
    N-->>V: 到达 (result)
    V->>T: /target_medicine + /medicine_pick_enable
    T->>C: /color_position (药品位置)
    C->>C: PD 控制底盘对准 (/cmd_vel)
    C->>A: /arm_state=pick
    A->>A: 发布 joint_trajectory 抓取
    A-->>V: /medicine_pick_state=done, /arm_phase
    V->>N: Action navigate_to_pose → 病房门口→送药点
    N-->>V: 到达
    V->>A: /arm_state=rotate_put 放药
    A-->>V: 放置完成
    V->>R: /medicine_delivery_record 送药记录
    V->>N: Action navigate_to_pose → 返回护士站
    N-->>V: 到达, 任务结束
```

## 关键话题 / Action 对照表（写报告可引用）

| 通信名称 | 类型 | 发布方 → 订阅方 | 作用 |
|---|---|---|---|
| `navigate_to_pose` | Action (NavigateToPose) | voice_nav_control → nav2_qing | 自主导航目标点 |
| `/goal_pose` | Topic (PoseStamped) | voice_nav_control → nav2_qing | 导航目标位姿 |
| `/scan` | Topic (LaserScan) | p2117_ros → jie_ware/nav2 | 激光雷达扫描 |
| `/camera/color(depth)/image_raw` | Topic (Image) | astra_camera → visual_tracker | 彩色/深度图像 |
| `/color_position` | Topic (SixArmPosition) | visual_tracker → 对准/抓取节点 | 药品像素位置与深度 |
| `/cmd_vel` | Topic (Twist) | nav2 / 对准节点 → communication_base | 底盘速度指令 |
| `/arm_state` | Topic (String) | voice_nav_control/car_location_color → 机械臂 | 机械臂动作指令，`pick` 由底盘对准节点触发，`rotate_put/no_msg` 由任务状态机触发 |
| `joint_trajectory` | Topic (JointTrajectory) | 机械臂节点 → communication_base | 关节轨迹执行 |
| `/medicine_pick_state`,`/arm_phase` | Topic (String) | 机械臂 → voice_nav_control | 抓取/阶段状态回传 |
| `/voice_nav_task` | Topic (String) | voice_nav_control → web | 任务状态 |
| `/medicine_delivery_record` | Topic (String) | voice_nav_control → web/onenet | 送药记录上报 |
| `map->odom_combined` | TF | jie_ware/lidar_loc | 全局定位变换 |
